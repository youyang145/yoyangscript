# 悠扬工具箱 — 开发者发布更新指南

## 一、上传哪些文件到 GitHub

把整个 yoyangscript 文件夹拖入上传，但确保以下内容已排除：

```
userdata/          ← 已移走，用户数据不上传
__pycache__/       ← 已删除，Python 缓存
```

**上传清单（24 个文件）：**

```
.gitignore
```

## 二、发布新版本（每次更新）

### 第 1 步：改版本号

编辑 config.py，改 VERSION：

```python
VERSION = "2.0.1"    # 比上次大即可
```

### 第 2 步：网页上传覆盖

1. 打开 https://github.com/youyang145/yoyangscript/upload/main
2. 把改过的文件拖进去
3. Commit message 写：v2.0.1 修复了XXX
4. 点 Commit changes

### 第 3 步：创建 Release

1. 打开 https://github.com/youyang145/yoyangscript/releases
2. 点 Draft a new release
3. Tag：v2.0.1（版本号前加 v）
4. Title：v2.0.1 - 简短说明
5. Describe 写更新日志（支持 Markdown）：

```
## 修复
- 修复日志编码错误

## 新增
- 一键自动更新
```

6. 点 Publish release

## 三、用户端的更新流程

```
启动 → 后台检查 GitHub Release → 有新版本？
  ├─ 否 → 不打扰
  └─ 是 → 弹窗
         ├─ [立即更新] → 自动下载 → 解压 → 覆盖程序文件
         │                 userdata/ 和 config.py 不会动
         │                 提示重启生效
         ├─ [取消]
         └─ [手动下载] → 打开 GitHub Releases 页面
```

---

## 四、版本号规则（SemVer）

| 数字 | 何时改 | 示例 |
|------|--------|------|
| 主版本 | 大改动、不兼容 | 2.0.0 → 3.0.0 |
| 次版本 | 新功能 | 2.0.0 → 2.1.0 |
| 修订版 | Bug 修复 | 2.0.0 → 2.0.1 |

## 五、GitHub API 限流

- 匿名 60 次/小时
- 下载 zip 不耗配额
- 检查更新有 10 分钟缓存
- 限流时会提示用户，不会崩溃
